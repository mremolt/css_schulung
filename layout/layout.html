Ein Layout ohne Tabellen

Sie sehen hier ein Layout, das bin hinunter zum IE7 zuverlässig funktioniert. Es basiert auf einem zentrierten Container-DIV mit fester Breite (1200px).

 

Anstatt die Breite jeder Spalte manuell einzugeben, wird ein vorgegebenes Grid aus 12 Spalten verwendet, man muss alsso nur noch sagen, dass dieser DIV 8 Spalten breit sein soll. Das erleichtert erheblich die Übersichtlichkeit und Wartbarkeit des CSS und verhindert inkonsistente Breiten im Layout. Dank box-sizing: border-box; zerstören auch paddings in den inneren Spalten das Layout nicht.

Die einzelnen Spalter jeder Zeile floaten zu einander (float: left) und es wurde ein fester Abstand zwischen den Spalten eingebaut. Es ist auch möglich, in eine Spalte wieder Spalten zu anzulegen, also verschachtelte Layouts zu bauen.
Wenn man bedenkt, dass hierfür nicht einmal 100 Zeilen CSS notwendig sind, sollte die Zeit der Tabellen-Layouts nun endgültig vorbei sein. Fertige Grid-Systeme wie z.B. Twitter Bootstrap bieten natürlich noch mehr Komfort und Funktionen.
